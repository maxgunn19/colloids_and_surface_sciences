# Git Quick Reference: Syncing Work Between Laptops

This guide covers the basic commands to keep your project synchronized across multiple computers 
using a central remote repository like GitHub.

The key is to always **pull** the latest changes from the remote before you **push** your 
new work up.

---

## 💻 Scenario A: Saving & Uploading Work (e.g., on Laptop 1)

Follow these steps when you've finished working on one computer and want to save your progress to the cloud.

### Step 1: Check for Remote Changes (Crucial First Step!)
Always start by pulling to make sure you have the most up-to-date version of the repository. 
This prevents conflicts.


git pull origin main

### Step 2: Stage Your Changes
This command adds all modified and new files to the "staging area," getting them ready to be saved.

git add .


### Step 3: Commit Your Changes
This saves a snapshot of your staged files. The message should be a short description of the work you did.

git commit -m "Your descriptive message here"


### Step 4: Push Your Work to the Cloud
This uploads your committed changes to your remote repository on GitHub.

git push origin main


---

## 🖥️ Scenario B: Downloading Work (e.g., on Laptop 2)

Use this command when you start working on your second computer to download all the changes 
you just pushed from your first computer.

git pull origin main


---

## ✨ The Golden Rule

Before you start working or before you push new changes, **always run `git pull`**. 
This simple habit will save you from most common sync issues.
